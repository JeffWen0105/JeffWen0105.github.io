網路介紹： OSI與 TCP/IP 模組 、 網路設備 、網路服務等基礎介紹.

網路基礎

OSI VS TCP/IP 模組比較

OSI 模組區分為七層架構， TCP/IP 模組為四層架構 ， 各有各的優點，最終各取的優點成為下圖最右邊架構，保留 OSI 模組的第一層（實體層）及第二層（資料連結層），合併 TCP/IP 第三層（網路層）、第四層（傳輸層）、第五層（應用層）。

Note
現在主流都以 TCP/IP 新的模組架構稱呼各層級

網路設備

  • 集線器(HUB)
  • 橋接器(Bridge)
  • 交換器(Switch)
  • 路由器(Router)

集線器(HUB)

集線器利用雙絞（RJ45）線將同一辦公室內的電腦接上，方便各電腦傳輸入（出）資料，及集中管理（星狀拓樸），屬於 OSI 模組第一層（實體層）網路設備。

  • 優點

    • 容易加入或移除電腦設備。

  • 缺點

    • 星狀拓樸如一個 Hub 設備故障，連接的電腦均會無法使用網路。
    • 一台電腦發送訊號，所有電腦都會收到（使用廣播方式），增加頻寬損耗。

橋接器(Bridge)

橋接器主要應用為集線器擴增網路區段，上述說到集線器會透過廣播發送訊號，如果一個區域電腦過多且同時發送訊息，將會大幅增加網路交通問題，故使用橋接器區分不同網路區段，屬於 OSI 模組第二層（資料連結層）網路設備，透過學習不同區段的 MAC 位址，轉送及篩選廣播訊框，舒緩網路交通瓶頸。

  • 優點

    • 增加網路區段。
    • 減少過多電腦的交通瓶頸。
    • 連接不同網路設備。

  • 缺點

    • 同網路區段還是需藉由集線器整合，還是會有頻寬損耗問題。
    • 產生網路風暴時無法消除。
Note
網路風暴是指因人為接錯線路，導致大量廣播在網路迴路內，使正常網路信號無法流通。

交換器(Switch)

交換器具備橋接器及集線器的功能，會自動記錄網路上電腦硬體的 MAC 位址，可以大幅降低網路傳輸頻寬瓶頸，目前大多數都改用交換器取代集線器及橋接器等網路設備，屬於 OSI 模組第二層（資料連結層）網路設備。

  • 優點

    • 自行學習MAC位置
    • 電腦等網路設備不需設定可以直接支援交換器。
    • 可以使用V-LAN切割網段。
    • 較專業設定具備 Spanning Tree 演算法，可以降低網路風暴的影響。

  • 缺點

    • 囿於要篩選MAC會有一定的攻耗，越頂級交換器單價越高。

路由器(Router)

路由器主要應用在不同連接不同的網段、不同的網路架構，路由器可以隔絕網路廣播，並選擇最佳路徑傳遞資訊，路由器中有路由表可以記錄網路位址、下一站路由器的位址等，屬於 OSI 模組第四層（網路層）網路設備。

  • 優點

    • 提供不同網段之間的訊框路由。
    • 隔絕廣播訊框傳遞。

  • 缺點

    • 路由器及電腦設備均設定及規劃。

網路服務

  • DHCP
  • DNS
  • WWW

DHCP （Dynamic Host Configuration Protocol 動態主機設定協定)

DHCP 透過該服務可以讓用戶端的電腦可以自動取得IP，在DHCP上提供一個IP發放範圍，預設電腦會自動尋找DHCP服務，並向該DHCP服務（伺服器）租賃IP，租賃IP的時效可以透過設定改變，每達到租約一半時間內，客戶端電腦將會發送請求給伺服器再次延長IP租約。

  • 發送租約包含：

    • IP位址
    • 子網路遮罩 (Subnet Mask)
    • 預設閘道 (Default GatWay)
    • 網域名稱 (Domain Name)
    • 網域名稱伺服器位址

DNS ( Domain Name System 網域名稱系統 )

DNS 主要功能為記錄IP及對應的網域名稱，網路通訊均透過 IP，惟沒有意義的 IP 數字對於人類難以記憶，相對於有意義的英文名稱比較，故網路名稱系統將存放 IP 網域名稱對照，例如 google.com　對應的IP為 172.217.163.46 ，所以如果想要有自己在網際網路上的域名需至對應的公司進行註冊（年費）才行。

WWW ( World Wide Web 全球資訊網 )

網際網路相信大家都不陌生，網頁主要是文字檔案格式化和超文件標示語言（HTML），透過該工具讓全世界的人都能在網際網路上互動。

  • 常用的網頁伺服器:

    • IIS (Internet Information Services)
    • Apache (Apache HTTP Server)
    • Nginx
